ISM2 Reverse-Engineered File Specification


Files are in Little endian
All offsets found in the file are relative to the beginning of the file
Every number with a "0x" prefix is represented in hexadecimal, otherwise in decimal

Header

Contains general information about the file
Offset Length Type Description
0x00 4 Magic Number "ISM2" in ASCII
0x04 4 (2*2)? u32 (2*u16)? Version ? (0x0101 or 0x0100)
0x08 8 Padding (0x00)
0x10 4 u32 Entire File Size
0x14 4 u32 Number of sections
0x18 8 Padding (0x00)
0x20 Variable Offsets to sections in the file in the order they are layed out
Multiple entries
Offset Length Type Description
0x00 4 u32 Section Magic Number
0x04 4 u32 Offset to section
Sections seems to follow this order:
  1. 0x21 - Strings Index Table
  2. 0x2E - Texture Definition
  3. 0x61
  4. 0x62
  5. 0x03 - Joint Definitions
  6. 0x32 - Joint Extra Information
  7. 0x0B - 3D Data Buffers

0x44 - Buffer

This sub-section appears multiple times...
Offset Length Type Description
0x00 4 u32 Magic number (0x44)
0x04 4 u32 0x20
0x08 4 u32 Number of entries
(note that, for some reason, the Inverse Bind Matrix buffer has this set to a number 16 times bigger than the actual amount)
0x0C 4 u32 0x00
0x10 4 u32 0x05 | 0x0C
0x14 4 u32 0x01 | 0x10
0x18 4 u32 0x00 | 0x10
0x1C 4 u32 0x00
0x20 Variable Buffer data
The buffers content seems to follow this order:
  1. Indexes to String table for naming each bone (stride of 2 bytes)
  2. Inverse bind matrices (stride of 64 bytes)

0x0B - 3D Data Buffers

Contains the general information on how the model looks like in 3D space.
Contains multiple sub-sections
Offset Length Type Description
0x00 4 u32 Magic Number (0x0B)
0x04 4 u32 0x0C
0x08 4 u32 Number of sub-sections (should be 1)
0x0C 4 u32 Offset to 0x0A sub-section

0x0A

Offset Length Type Description
0x00 4 u32 Magic Number (0x0A)
0x04 4 u32 0x20
0x08 4 u32 Number of sub-sections
0x0C 4 u32 Index to String ("model00Shape2", "accessoryShape")
0x10 4 u32 Same as previous
0x14 4 u32 0x00
0x18 4 u32 0x01 | 0x02
0x1C 4 u32 0x01 | 0x03
0x20 Variable Multiple entries
Offset Length Type Description
0x00 4 u32 Offset to sub-sections 0x59, 0x46, 0x6E
They are always listed in the structure above in the order they appear in the file.
Each sub-section start might or might not be aligned

0x59 - Vertices

Contains information about each vertex in order in the file.
Accessories have a single section, while characters (or anything skinned) have two
Offset Length Type Description
0x00 4 u32 Magic Number (0x59)
0x04 4 u32 0x1C
0x08 4 u32 Number of sub-sections
0x0C 4 u32 0x01 | 0x03
0x10 4 u32 Amount of vertices
0x14 4 u32 Buffer Stride ? (Always 0x20)
0x18 4 u32 0x00
0x20 Variable Offsets to sub-sections
Multiple entries
Offset Length Type Description
0x00 4 u32 Offset to sub-sections 0x00, 0x02, 0x0E, 0x03 | 0x07, 0x01
Sub-sections referenced in 0x20 are attributes found in the buffer
This data looks like:
Offset Length Type Description
0x00 4 u32 Magic Number (0x00, 0x02, 0x0E, 0x03...)
0x04 4 u32 0x04
0x08 4 u32 0x03
0x0C 4 u32 Always 0x20, probably how far to jump for next set of data (same as buffer stride)
0x10 4 u32 0x1C
0x14 4 u32 Offset to buffer
It seems like the following layout is always used in all model files (buffer stride of 0x20 bytes):
Offset Length Type Description
0x00 4 f32 Position X Coordinate
0x04 4 f32 Position Y Coordinate
0x08 4 f32 Position Z Coordinate
0x0C 2 f16 Normal X Coordinate
0x0E 2 f16 Normal Y Coordinate
0x10 2 f16 Normal Z Coordinate
0x12 2 f16 Texture U Coordinate
0x14 2 f16 Tangent ? X Coordinate
0x16 2 f16 Tangent ? Y Coordinate
0x18 2 f16 Tangent ? Z Coordinate
0x1A 2 f16 Texture V Coordinate
0x1C 2 u16 Always 0xFF (padding ?)
0x1E 2 u16 Always 0xFF (padding ?)
This seems to be the general pattern for the second buffer found in characters (stride of 0x20 bytes):
Offset Length Type Description
0x00 1 u8 First influencing Joint
0x01 1 u8 Second influencing Joint
0x02 1 u8 Third influencing Joint
0x03 1 u8 Fourth influencing Joint
0x04 4 f32 Weight for First Joint
0x08 4 f32 Weight for Second Joint
0x0C 4 f32 Weight for Third Joint
0x10 4 f32 Weight for Fourth Joint
0x14 12 0x00 Padding
