ISM2 Reverse-Engineered File Specification


Files are in Little endian
All offsets found in the file are relative to the beginning of the file
Every number with a "0x" prefix is represented in hexadecimal, otherwise in decimal

Header

Contains general information about the file
Offset Length Type Description
0x00 4 Magic Number "ISM2" in ASCII
0x04 4 (2*2)? u32 (2*u16)? Version ? (0x0101 or 0x0100)
0x08 8 Padding (0x00)
0x10 4 u32 Entire File Size
0x14 4 u32 Number of sections
0x18 8 Padding (0x00)
0x20 Variable Offsets to sections in the file in the order they are layed out
Multiple entries
Offset Length Type Description
0x00 4 u32 Section Magic Number
0x04 4 u32 Offset to section
Sections seems to follow this order:
  1. 0x21 - Strings Index Table
  2. 0x2E
  3. 0x61
  4. 0x62
  5. 0x03 - Joint Definition
  6. 0x32 - Joint Extra Information
  7. 0x0B - 3D Data Buffers

0x21 - Strings Index Table

Contains strings of text used throughout the file.
Offset Length Type Description
0x00 4 u32 Magic number (0x21)
0x04 4 u32 0x0C
0x08 4 u32 Number of entries
0xC Variable Offsets to every string entry in the following buffer
Multiple entries
Offset Length Type Description
0x00 4 u32 Offset to entry
... Variable Text buffer
Multiple entries
Offset Length Type Description
0x00 Variable String of text
... 1 u8 ASCII NUL Separator (0x00)

There is one of these sections per joint

0x00 4 u32 Magic number (0x05)

0x08 4 u32 Number of sub-sections

0x5B - Offsets to attributes

Offset Length Type Description
0x00 4 u32 Magic number (0x5B)
0x04 4 u32 0x0C
0x08 4 u32 Number of offsets
0x0C Variable Multiple entries
Offset Length Type Description
0x00 4 u32 Offset to attribute ( 0x14, 0x67, 0x68, 0x69)
0x14 - Transform
Describes the Transform of the joint relative to its parent
Offset Length Type Description
0x00 4 u32 Magic number (0x14)
0x04 4 u32 0xC2 | 0xC1 | 0xB4 | 0xBC | 0x4B | 0xAD
0x08 4 f32 X Transform
0x0C 4 f32 Y Transform
0x10 4 f32 Z Transform
0x67 - Euler Roll Rotation
Sets the Roll of this joint
Offset Length Type Description
0x00 4 u32 Magic number (0x67)
0x04 4 u32 0xD9
0x08 4 u32 1f32, indicates Roll in the matrix
0x0C 4 u32 0f32, indicates Pitch in the matrix
0x10 4 u32 0f32, indicates Yaw in the matrix
0x14 4 u32 Angle in degrees
0x68 - Euler Pitch Rotation
Sets the Pitch of this joint
Offset Length Type Description
0x00 4 u32 Magic number (0x67)
0x04 4 u32 0xD9
0x08 4 u32 0f32, indicates Roll in the matrix
0x0C 4 u32 1f32, indicates Pitch in the matrix
0x10 4 u32 0f32, indicates Yaw in the matrix
0x14 4 u32 Angle in degrees
0x69 - Euler Yaw Rotation
Sets the Yaw of this joint
Offset Length Type Description
0x00 4 u32 Magic number (0x67)
0x04 4 u32 0xD9
0x08 4 u32 0f32, indicates Roll in the matrix
0x0C 4 u32 0f32, indicates Pitch in the matrix
0x10 4 u32 1f32, indicates Yaw in the matrix
0x14 4 u32 Angle in degrees

0x32 - Joint Extra Information

This section is only present in character files, and seems to only contain IDs to the String Table and Inverse Bind Matrices for each joint.
Offset Length Type Description
0x00 4 u32 Magic number (0x32)
0x04 4 u32 0x14
0x08 4 u32 0x01
0x0C 4 u32 0x00
0x00 4 u32 0x00
0x00 4 u32 Offset to next sub-section

0x31

Offset Length Type Description
0x00 4 u32 Magic Number
0x04 4 u32 0x14
0x08 4 u32 0x01
0x0C 4 u32 0x30 | 0x31 | 0x32
0x10 4 u32 0x31 | 0x32 | 0x33 (seems to always be +1 of the previous field)
0x14 4 u32 Offset to next sub-section

0x30

Offset Length Type Description
0x00 4 u32 Magic number
0x04 4 u32 0x54
0x08 4 u32 0x02
0x0C 4 u32 0x1B | 0x1C
0x10 68, 70, ... Some kind of data
0x803F appears a lot but is not aligned on 4 bytes
(0x3F800000 is 1 in f32, 0x3F80 is 1.875 in f16)
After data 4 u32 Offset to next sub-section
After data +4 4 u32 Offset to other next sub-section

0x44 - Buffer

This sub-section appears multiple times...
Offset Length Type Description
0x00 4 u32 Magic number (0x44)
0x04 4 u32 0x20
0x08 4 u32 Number of entries
(note that, for some reason, the Inverse Bind Matrix buffer has this set to a number 16 times bigger than the actual amount)
0x0C 4 u32 0x00
0x10 4 u32 0x05 | 0x0C
0x14 4 u32 0x01 | 0x10
0x18 4 u32 0x00 | 0x10
0x1C 4 u32 0x00
0x20 Variable Buffer data
The buffers content seems to follow this order:
  1. Indexes to String table for naming each bone (stride of 2 bytes)
  2. Inverse bind matrices (stride of 64 bytes)

0x0B - 3D Data Buffers

Contains the general information on how the model looks like in 3D space.
Contains multiple sub-sections
Offset Length Type Description
0x00 4 u32 Magic Number (0x0B)
0x04 4 u32 0x0C
0x08 4 u32 Number of sub-sections (should be 1)
0x0C 4 u32 Offset to 0x0A sub-section

0x0A

Offset Length Type Description
0x00 4 u32 Magic Number (0x0A)
0x04 4 u32 0x20
0x08 4 u32 Number of sub-sections
0x0C 4 u32 0x0D | 0x1B | 0x1C ("model00Shape2")
0x10 4 u32 Same as previous
0x14 4 u32 0x00
0x18 4 u32 0x02
0x1C 4 u32 0x01 | 0x03
0x20 Variable Multiple entries
Offset Length Type Description
0x00 4 u32 Offset to sub-sections 0x59, 0x46, 0x6E
They are always listed in the structure above in the order they appear in the file.
Each sub-section start might or might not be aligned

0x59 - Vertices

Contains information about each vertex in order in the file.
Accessories have a single section, while characters can have multiple...
(probably for specifying the index of the joint the vertex is connected to)
Offset Length Type Description
0x00 4 u32 Magic Number
0x04 4 u32 0x1C
0x08 4 u32 Number of sub-sections
0x0C 4 u32 0x01 | 0x03
0x10 4 u32 Amount of vertices
0x14 4 u32 Buffer Stride ? (Always 0x20)
0x18 4 u32 0x00
0x20 Variable Offsets to sub-sections
Multiple entries
Offset Length Type Description
0x00 4 u32 Offset to sub-sections 0x00, 0x02, 0x0E, 0x03 | 0x07, 0x01

0x03 - Data

Offset Length Type Description
0x00 4 u32 Magic Number (0x03)
0x04 4 u32 0x04
0x08 4 u32 0x03
0x0C 4 u32 0x20
0x10 4 u32 0x1C
0x14 4 u32 Offset to buffer
Padding to align on 16 bytes
... Variable Buffer containing vertices data
It seems like the following layout is always used in all model files (buffer stride of 0x20):
Offset Length Type Description
0x00 4 f32 Position X Coordinate
0x04 4 f32 Position Y Coordinate
0x08 4 f32 Position Z Coordinate
0x0C 2 f16 ]-1; 1[ (Normal X ?)
0x0E 2 f16 ]-1; 1[ (Tangent X ?)
0x10 2 f16 ]-1; 1[ (Bi-normal X ?)
0x12 2 f16 Texture U Coordinate
0x14 2 f16 [-1; 1] (Normal Y ?)
0x16 2 f16 ]-1; 1[ (Tangent Y ?)
0x18 2 f16 [-1; 1] (Bi-normal Y ?)
0x1A 2 f16 Texture V Coordinate
0x1C 2 u16 Always 0xFF (padding ?)
0x1E 2 u16 Always 0xFF (padding ?)

0x46 - Faces

Contains each points of the faces defining the shape of the model.
It seems to always be triangles.
Offset Length Type Description
0x00 4 u32 Magic Number (0x46)
0x04 4 u32 0x1C
0x08 4 u32 Amount of sub-sections
0x0C 4 u32 Points to names like 'blinn2SG', '_blinn3SG', ...
0x10 4 u32 0x00
0x10 4 u32 0x04
0x18 4 u32 Number of faces
0x1C Variable Offsets to sub-sections
Multiple Entries
Offset Length Type Description
0x00 4 u32 Offset to sub-section 0x45, 0x6E

0x45

Offset Length Type Description
0x00 4 u32 Magic Number (0x45)
0x04 4 u32 0x14
0x08 4 u32 Amount of IDs in total in the buffer (should be Amount of faces * 3)

0x10 4 u32 0x00

Multiple entries

0x02 2 u16 Vertex 2 ID
